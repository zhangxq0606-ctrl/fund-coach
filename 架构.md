# Fund Coach 架构总览

> 投资哲学详见 `产品理念.md`，UI/UX 设计详见 `产品设计.md`。

---

## 一、产品定位

- **定位**：AI 原生的基金定投分析工具，基于**核心-卫星**策略提供个性化建议
- **目标用户**：学生及普通投资者
- **核心原则**：
  - 核心仓（宽基为主 70-80%）+ 卫星仓（行业主题 20-30%）
  - 核心仓弹性定义：用户长期看好的、覆盖多个行业的指数也可纳入核心仓
  - 不涉及资金交易、不绑定银行卡、不要求转入资金
  - 纯分析顾问，无需登录注册，开箱即用

---

## 二、技术选型

| 层级 | 选型 | 说明 |
|------|------|------|
| **前端** | HTML + TailwindCSS（CDN）+ 原生 JavaScript | SPA，hash 路由，4 页（首页/持仓管理/投资画像/对话页），浅色主题 |
| **后端** | Python FastAPI（一体化） | 托管前端静态文件 + API 接口，单服务部署 |
| **AI 模型** | DeepSeek Chat（V3） | 支持 Function Calling，上下文 64K |
| **SDK** | OpenAI Python SDK | `base_url=https://api.deepseek.com/v1` |
| **实时数据** | 天天基金 JS 接口（主）+ AKShare（备） | 双数据源容灾 |
| **存储** | 浏览器 localStorage | 持仓列表、对话历史、画像数据 |
| **部署** | Railway / Render | 支持 Python，自动 HTTPS |

## 三、架构设计理由

### 为什么选 FastAPI 一体化？

AKShare 是 Python 库，无法在纯前端/Serverless JS 环境中运行。FastAPI 一体化链路最短：

```
用户 → FastAPI → DeepSeek API → FastAPI 执行数据查询 → DeepSeek 最终回答 → 用户
```

### 为什么选一体化托管而非前后端分离？

小范围使用场景下，一个域名、一次部署、无跨域问题，开发调试最简单。

## 五、核心链路

```
用户打开网页
  → 从 localStorage 加载持仓列表、对话历史、画像

用户在对话页提问
  → 前端构造请求体：
    {
      "holdings": [{ "fund_code": "005827", "fund_name": "易方达蓝筹", "amount": 1000, "frequency": "monthly" }],
      "user_profile": { "risk_level": "稳健型" },
      "conversation_history": [...],
      "new_message": "用户问题"
    }
  → POST /api/chat

后端处理流程：
  1. 构造 DeepSeek API 调用（含 Function Calling + System Prompt）
  2. DeepSeek 判断是否需要实时数据或更新画像
  3. 若需要数据 → 返回 Function Call（如 get_fund_nav("005827")）
  4. 后端执行数据查询（天天基金 JS 优先 → AKShare 兜底）
  5. 将结果附加到消息中，再次调用 DeepSeek
  6. 若 DeepSeek 调用 update_user_profile → 后端校验 confidence ≥ 0.6，将结果附入消息再次调用 DeepSeek 生成最终回答
  7. 返回响应：`{ "reply": "...", "profile_update": {...} }`（profile_update 可选）

前端展示回答；若响应含 profile_update，写入 localStorage 的 user_profile key
```

## 八、Function Calling 定义

### get_fund_nav（核心）

获取某只基金的最新净值和近期收益表现。

**入参：** `{ "fund_code": "005827" }`

**成功返回：**
```json
{
  "fund_code": "005827",
  "fund_name": "易方达蓝筹精选混合",
  "latest_nav": 2.1587,
  "nav_date": "2026-06-09",
  "nav_change_pct": -0.52,
  "one_month_return": 2.13,
  "three_month_return": -1.87,
  "ytd_return": 5.42,
  "error": null
}
```

**失败返回：** `{ "fund_code": "005827", "error": "数据暂不可用，请稍后重试" }`

### update_user_profile

AI 将评测结果写入用户画像。

**出参（不涉及外部调用，后端直接返回给前端）：**
```json
{
  "risk_level": "稳健型",
  "investment_experience": "1-3年", 
  "investment_goal": "长期增值",
  "confidence": 0.8,
  "updated_at": "2026-06-11T12:00:00Z"
}
```

**后端响应格式扩展**（在原有 `/api/chat` 响应中增加 `profile_update` 字段）：
```json
{
  "reply": "根据你的回答，你的风险承受能力偏稳健...",
  "profile_update": {
    "risk_level": "稳健型",
    "investment_experience": "1-3年",
    "investment_goal": "长期增值",
    "confidence": 0.85
  }
}
```
前端收到 `profile_update` 后自动写入 localStorage 的 `user_profile` key。

**Function Calling 定义：**
| 属性 | 值 |
|------|-----|
| name | `update_user_profile` |
| description | "根据对话中收集到的用户信息，更新用户投资画像。收集足够信号（3-5个维度）后再调用，不要过早调用。" |
| parameters | `{risk_level, investment_experience, investment_goal, confidence}` |

**risk_level 可选值：** `保守型`, `稳健型`, `平衡型`, `进取型`, `激进型`
**investment_experience 可选值：** `刚接触`, `1年以内`, `1-3年`, `3-5年`, `5年以上`
**investment_goal 可选值：** `资金保值`, `长期增值`, `收益最大化`
**confidence 范围：** 0.0 ~ 1.0（0.6 以下后端丢弃不更新）

### get_index_valuation（新增）

获取指定指数的估值分位数据（PE/PB 及历史百分位），用于判断"估值贵不贵"。

**入参：** `{ "index_code": "000300", "index_name": "沪深300" }`

**index_code 对照表：**
| 指数 | code |
|------|------|
| 沪深300 | 000300 |
| 中证500 | 000905 |
| 中证A500 | 000510 |
| 创业板指 | 399006 |
| 科创50 | 000688 |
| 上证指数 | 000001 |

**返回示例：**
```json
{
  "index_code": "000300",
  "index_name": "沪深300",
  "pe": 12.5,
  "pb": 1.35,
  "pe_percentile": 45.2,
  "pb_percentile": 38.7,
  "pe_50th": 13.8,
  "data_date": "2026-06-10",
  "error": null
}
```

**数据源：** AKShare `index_valuation_hist_em`（东方财富指数估值历史数据）

### get_macro_data（新增）

获取宏观经济指标数据，辅助时局分析。

**入参：** `{ "indicator": "pmi" }`（pmi / cpi / shibor 之一）

**返回示例（pmi）：**
```json
{
  "indicator": "pmi",
  "latest_value": 50.8,
  "latest_month": "2026-05",
  "trend": "连续3个月处于扩张区间",
  "data": [
    {"month": "2026-01", "value": 49.1},
    {"month": "2026-02", "value": 49.9},
    {"month": "2026-03", "value": 50.5},
    {"month": "2026-04", "value": 50.4},
    {"month": "2026-05", "value": 50.8}
  ],
  "error": null
}
```

**数据源：** AKShare 宏观经济接口（`macro_china_pmi` / `macro_china_cpi` / `rate_interbank`）

## 九、关键约束

1. **API Key 安全**：前端不暴露 DeepSeek Key，全部走后端代理，通过环境变量加载
2. **合规声明**：每个页面底部小字"本工具仅供学习交流，不构成投资建议"
3. **严格模式**：数据不可用时，不降级到大模型知识，明确告知用户数据暂不可用
4. **无状态后端**：每次请求携带完整上下文，后端不存状态
5. **浅色主题**：蓝灰调，移动端优先，max-width 约束
6. **System Prompt 红线**：AI 不主动推荐行业基金、不提供买卖时点建议
7. **用户画像动态适配**：通过对话自然推断用户水平调整回答深度，而非静态表单分级

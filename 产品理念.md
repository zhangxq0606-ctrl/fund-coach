## 一、核心投资哲学

**核心-卫星策略：核心压舱，卫星探路。**

### 定投（DCA）
- 定期定额投入，平滑市场波动
- 不预测短期涨跌，靠时间积累复利
- 适合学生/普通投资者：不需要专业知识，不需要盯盘

### 核心-卫星框架

```
┌─────────────────────────────────────┐
│         你的基金定投组合             │
│                                     │
│  ┌──── 核心仓（70-80%） ────────┐  │
│  │  宽基指数基金                 │  │
│  │  - 沪深300 / 中证A500        │  │
│  │  - 中证500 / 中证1000        │  │
│  │  - 纳斯达克100 / 标普500     │  │
│  │  → 买整个市场，长期持有       │  │
│  └─────────────────────────────┘  │
│                                     │
│  ┌──── 卫星仓（20-30%） ────────┐  │
│  │  行业/主题基金（适量）        │  │
│  │  - 科技 / 医药 / 新能源      │  │
│  │  - 消费 / 半导体等           │  │
│  │  → 锻炼前瞻判断力，小仓试错   │  │
│  └─────────────────────────────┘  │
│                                     │
│  AI 分析两个层面：                   │
│  ① 核心仓的各宽基配比是否合理？     │
│  ② 卫星仓的方向是否需要调整？       │
└─────────────────────────────────────┘
```

### 核心仓
- 宽基指数基金，占主体（建议 70-80%）
- 分散于不同市场（A股、港股、美股等）
- 不赌单一国家、单一风格，长期持有
- 时局变化时调整各宽基的**定投额占比**（如 A股 vs 美股）

### 卫星仓
- 行业/主题基金，占小部分（建议 20-30%）
- 目的是**培养学生对产业趋势的前瞻判断力**
- 小仓试错，亏了不伤筋骨，赚了有成就感
- AI 不主动推荐行业基金，但如果用户持有，帮助分析其逻辑是否成立

### 核心原则
| 原则 | 说明 |
|------|------|
| 定投为主 | 不预测短期，靠时间复利 |
| 核心大于卫星 | 宽基是主体，行业是配角 |
| 不频繁调整 | 只在确定性较高的中长期趋势拐点才建议变动 |
| 灵活处理 | 工具不预设用户持仓，用户买什么就分析什么 |

### 核心仓的弹性定义

核心仓不限于传统宽基（沪深300、中证500等）。用户长期看好的、定义了国家经济主线的指数（如科创创业50），也可纳入核心仓。关键判断标准：
- 用户是否计划 **长期持有**（3年以上）
- 该指数是否覆盖 **多个行业**（虽集中但不单一行业）
- 用户的定投资金是否 **持续投入**

> 系统应尊重用户对"核心"的定义，在此基础上优化建议，而非死板按标签分类。

### 用户画像的动态适配

不同用户需要不同深度的指导：
- **小白用户**：侧重基础教育和理念普及，引导理解核心-卫星策略
- **有经验用户**：侧重配比优化和时局分析，尊重其已有认知
- **系统处理方式**：通过对话自然推断用户水平，调整回答深度，而非静态表单分级

## 二、工具定位

**定投核心-卫星智能教练**，不是"选股工具"，不是"行业轮动工具"。

### 对学生群体的价值
1. **教育意义**：帮助学生理解定投 + 核心-卫星策略的长期价值
2. **陪伴意义**：在市场下跌时给信心，在市场狂热时给冷静
3. **分析意义**：判断当前时局下核心仓配比和卫星仓方向是否需要调整

### AI 的分析框架
- 不预设用户应该买什么
- 用户持仓中包含什么，AI 就分析什么
- 区分核心仓（宽基）和卫星仓（行业），给出不同层面的建议
- 对卫星仓：不主动推荐，但如果用户持有，帮助评估趋势逻辑
- 对核心仓：分析各宽基的估值水平、市场环境，建议配比调整

### 引导式分析（核心定位）
AI 的角色是**分析引导员**，不是投资顾问。每次分析按三层结构：

1. **诊断结论**：给出明确判断（如"组合集中度偏高"），不模棱两可，但不输出具体操作指令
2. **关注引导**：告诉用户关注什么信息源、什么时间节点、什么事件（如"关注月底PCE数据"）
3. **回访入口**：留下开放性问题，形成"分析→关注→再分析"的闭环

工具的核心价值在于帮用户"看明白现状、知道该关注什么"，而不是替用户做决策。

### 时局解读的定位
- 不是"行业赛道评分"
- 而是：**当前宏观环境下，你的核心-卫星结构是否合理？**
  - 宽基之间（A股/美股/港股）的配比需要调整吗？
  - 卫星仓的方向（科技/消费/医药等）是否符合产业趋势？
  - 整体风险暴露是否与用户画像匹配？

## 三、不做清单（产品红线）

1. ❌ 不推荐个股
2. ❌ 不提供买卖时点建议（"明天买入"、"下周卖出"这类）
3. ❌ 不涉及资金交易、不绑定银行卡
4. ❌ 不展示K线图、技术指标
5. ❌ 不鼓励把卫星仓占比提到不合理水平（超过 30-40%）
6. ❌ 不在用户没持有行业基金的情况下主动推荐行业基金

### 可以做但要注意的
- ✅ 用户持有行业基金 → 帮助分析趋势逻辑 ✅
- ✅ 用户问"现在该关注哪些方向" → 可以从产业趋势角度给中性分析，但提示"卫星仓小仓试错" ✅
- ✅ 时局有明确拐点信号 → 建议调整核心仓宽基配比 ✅

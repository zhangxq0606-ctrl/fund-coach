# Fund Coach — 产品设计稿（UI/UX 定稿）

> 更新时间：2026-06-11
> 本文件记录前端 UI/UX 全部设计决策，供操盘手落地执行。

---

## 一、整体架构

**SPA，hash 路由**，共 4 个页面页：

| 路由 | 页面 | 说明 |
|------|------|------|
| `#/` | 首页 | 导航入口 + 投资理念教育窗口 |
| `#/holdings` | 持仓管理 | 展示/编辑持仓列表 |
| `#/profile` | 投资画像 | 评测/展示用户画像 |
| `#/chat` | 对话页 | AI 对话核心界面 + 时局解读入口 |

### 导航规则
- **无底部 Tab 栏**
- 首页使用按钮导航到子页面
- 子页面顶部有返回按钮（← 返回），回到首页
- 页面切换不丢失状态（localStorage 持久化）

### 设计要点
- 顶部 Logo + 标题，简洁
- 两个子页按钮（投资画像、持仓管理）并列或上下排列，图标 + 文字
- "进入对话"按钮更大、居中，视觉突出，引导用户进入核心功能
- 底部教育窗口默认折叠，点击展开。内容为纯知识科普，与用户持仓无关

### 教育窗口内容
```
📖 投资理念 · 入门知识

**什么是基金定投？**
定期定额投资基金，不预测短期涨跌，靠时间积累复利。

**核心-卫星策略**
核心仓（70-80%）：宽基指数基金，买整个市场，长期持有
卫星仓（20-30%）：行业/主题基金，小仓试错，培养前瞻判断力

**宽基 vs 行业基金**
宽基 = 覆盖全市场（如沪深300、中证500），风险分散
行业 = 聚焦单一赛道（如科技、医药），波动更大

**常见误区**
❌ 频繁买卖 → 增加成本，降低收益
❌ 追涨杀跌 → 高点买入低点卖出
❌ 行业基金当主仓 → 集中度高，风险大
```

## 三、持仓管理（#/holdings）

### 功能
- 展示用户所有持仓基金列表
- 每行显示：基金代码 + 基金名称 + 月定投金额
- 可编辑、可删除
- 底部 "+ 添加" 表单
- 顶部显示汇总信息：**持仓: N只基金 | 月投 X元 | 画像: XXX**
- 顶部有 ← 返回按钮

### 布局
```
┌──────────────────────────────────┐
│ ← 返回   持仓管理                │
│ 持仓: 2只基金 | 月投 1,500元     │
│ 画像: 稳健型                     │
├──────────────────────────────────┤
│ [代码] [名称] [月投] [×删除]    │
│ 013304 易方达科创创业50 每周20   │
│ 270042 广发纳斯达克100 每天4     │
│ ...                              │
├──────────────────────────────────┤
│ [+ 添加]                         │
│ [代码____] [名称____] [金额____] │
└──────────────────────────────────┘
```

### 字段说明
- 基金名称：用户手动输入简称（非自动获取）
- 定投频率：支持"每天/每周/每月"选择，后端统一换算为月投金额
- 数据存 localStorage，key: `fund_holdings`

## 四、投资画像（#/profile）

### 功能
- 展示当前画像信息（从 localStorage 读取）
- 提供"重新评测"按钮，点击后触发 AI 情境对话评测
- 展示画像更新时间

### 核心设计：AI Native 画像系统

**分层架构：**

| 层 | 触发条件 | 方式 |
|---|---|---|
| 初始基线 | 首次使用 / 手动点击[重新评测] | AI 情境对话 3-5 问，聊完生成基线 |
| 动态校正 | 对话中自然发生 | AI 累计信号，置信度达阈值才更新 |

**初始基线流程：**
1. 用户点[重新评测] → 跳转对话页
2. AI 用情境对话而非选择题，自然聊天：
   - "你投资基金多久了？"
   - "假设你的组合突然一个月跌了15%，你第一反应是什么？"
   - "你定投的钱多久内不会用到？"
   - "你更倾向于长期持有还是根据市场变化调整？"
3. AI 收集完回答后，调用 `update_user_profile` 写入 localStorage
4. 画像页展示最新结果

**首次进入提示：**
- 用户无画像且首次进入对话页时，AI 主动提示："你还没做投资评测，要不要先花2分钟做一下？这样我的分析会更准。"
- 用户拒绝也可以继续使用，默认为"稳健型"

### 布局
```
┌──────────────────────────────────┐
│ ← 返回   投资画像                │
├──────────────────────────────────┤
│ 当前画像                         │
│ 风险等级：稳健型                 │
│ 投资经验：1-3年                  │
│ 投资目标：长期增值               │
│ 最近更新：2026-06-11             │
│                                  │
│ [🔄 重新评测]                     │
│                                  │
│ 你的画像会随着对话自然优化，     │
│ 越用越准。                       │
└──────────────────────────────────┘
```

## 五、对话页（#/chat）

### 核心布局
```
┌──────────────────────────────────┐
│ ← 返回   Fund Coach              │
│                                  │
│ [📊 时局解读]  ← 小按钮         │  ← 固定在对话上方
│                                  │
│ ┌────────────────────┐          │
│ │ AI 回复气泡         │ →        │
│ └────────────────────┘          │
│              ┌─────────────────┐│
│              │ ← 用户消息气泡   ││
│              └─────────────────┘│
│                                  │
│ ┌──────────────────────────────┐│
│ │ 输入问题...           [发送] ││  ← 固定底部
│ └──────────────────────────────┘│
└──────────────────────────────────┘
```

### 按键说明
- **📊 时局解读**：点击后自动在输入框填入预设提示词（不直接发送），内容为：
  > "根据我的持仓，分析当前时局下我的核心-卫星结构是否合理，各宽基配比是否需要调整，以及我的卫星仓方向是否符合产业趋势。"
- 用户可修改后发送，也可直接按发送

### 气泡样式
- AI 气泡：左对齐，灰色背景（`#e9eef4`），圆角
- 用户气泡：右对齐，蓝色背景（`#2563eb`），白色文字
- 加载状态："思考中..."
- 输入框固定底部，不随对话滚动

## 六、全局样式

- **主题**：浅色，简洁专业
- **颜色**：蓝灰调，主色 `#2563eb`（蓝色）
- **字体**：系统字体栈 `-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif`
- **响应式**：移动端优先，PC 端 max-width 约束
- **免责声明**：每个页面底部小字 "本工具仅供学习交流，不构成投资建议"

---

## 七、数据存储（localStorage）

| key | 类型 | 说明 |
|-----|------|------|
| `fund_holdings` | `[{fund_code, fund_name, amount, frequency}]` | 持仓列表，frequency: "daily"/"weekly"/"monthly" |
| `chat_history` | `[{role, content}]` | 对话历史 |
| `user_profile` | `{risk_level, investment_goal, ...}` | 用户画像 |

---

## 八、页面状态与过渡

- 页面切换时，当前页面的输入框内容/滚动位置不保留（hash 切换重新渲染）
- 持仓管理和对话页的数据从 localStorage 读取，不依赖后端
- 网络请求失败时展示错误提示，不影响本地数据

---

## 九、与后端的关系

- 后端只提供 `/api/chat` 一个 API
- 前端所有页面渲染、路由、数据存储都在浏览器端完成
- `/api/chat` 接收 `{holdings, user_profile, conversation_history, new_message}`，返回 `{reply}`
